# Migration: v0.1.0 -> v0.2.0

## Version

- **From:** 0.1.0
- **To:** 0.2.0

## Breaking Changes

- TanStack packages are now pinned to ~1.114.3 via pnpm overrides. Version 1.167+ dropped vinxi support and will break the build. Do not remove or loosen these overrides.

## Migration Steps

Follow these steps in order. Run `pnpm check` and `pnpm typecheck` after completing all steps.

### 1. Pin TanStack packages via pnpm overrides

Add the `pnpm.overrides` block from construct's `package.json` into the instance's `package.json`. This pins 17 TanStack packages to ~1.114.3. If the instance already has a `pnpm.overrides` section, merge the entries without removing existing overrides.

### 2. Add `@tanstack/router-cli` as a dev dependency

Add `@tanstack/router-cli` to `devDependencies`. Then run `pnpm install`.

### 3. Add `tsr.config.json`

Copy `tsr.config.json` from construct into the instance root. It points the route tree generator at `app/routes/` with output to `app/routeTree.gen.ts`.

### 4. Add the `postinstall` script

In `package.json`, add `"postinstall": "tsr generate"` to the `scripts` block. Run `pnpm install` to trigger it and confirm `app/routeTree.gen.ts` is generated.

### 5. Update `.gitignore`

Merge the following entries into the instance's `.gitignore` if not already present:

- `node_modules/`
- `.output/`
- `.vinxi/`
- `dist/`
- `.env*`
- `routeTree.gen.ts`
- `app.config.timestamp_*`
- `.claude/`

### 6. Update `biome.json`

Add `app.config.timestamp_*` and `.claude/` to the `files.ignore` array in `biome.json`. Do not remove existing ignore entries.

### 7. Add error boundary and 404 handling to `app/routes/__root.tsx`

Add `notFoundComponent` and `errorComponent` to the root route. Copy the implementations from construct's `app/routes/__root.tsx`. If the instance has customized this file, integrate the components without overwriting existing layout code.

### 8. Add example server function to `app/routes/index.tsx`

If the instance has not customized `app/routes/index.tsx` beyond the scaffold default, replace it with construct's version which includes an example server function. If the instance has customized it, skip this step.

### 9. Add example utilities and tests

Copy the following files from construct into the instance:

- `app/utils/format.ts`
- `app/utils/format.test.ts`
- `app/components/Greeting.tsx`
- `app/components/Greeting.test.tsx`

Do not overwrite if the instance already has files at these paths.

### 10. Update scaffold script

Replace `scripts/scaffold.mjs` with construct's version. This adds AGENTS.md substitution (step 8).

### 11. Update AGENTS.md and docs

Propagate `AGENTS.md` and all files in `docs/agents/` from construct. This includes:

- Expanded API docs in `docs/agents/api.md`
- New dependency governance doc at `docs/agents/dependencies.md`

### 12. Update CI workflow

Replace `.github/workflows/ci.yml` with construct's version. Changes include frozen-lockfile, pnpm cache, and conditional E2E based on `CI_E2E_DATABASE_URL` secret.

### 13. Add validate-template CI workflow

Copy `.github/workflows/validate-template.yml` from construct.

### 14. Add PR template

Copy `.github/pull_request_template.md` from construct.

### 15. Add preflight script

Add `"preflight": "pnpm check && pnpm typecheck && pnpm test run"` (or the current construct equivalent) to the `scripts` block in `package.json`.

### 16. Update `.construct` version

Set `constructVersion` to `"0.2.0"` in the instance's `.construct` file.

### 17. Verify

Run `pnpm check`, `pnpm typecheck`, and `pnpm test run`. Fix any issues before moving on.
