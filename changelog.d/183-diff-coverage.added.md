- `[manual]` Added an always-on diff-coverage CI gate that hard-fails every PR when its added/changed executable lines are covered below 80% (`.github/scripts/check-diff-coverage.mjs`, issue #183). Coverage is measured only on the lines the PR touches, so legacy untested code is never retroactively failed. With the `CI_E2E_DATABASE_URL` secret present the job runs the full unit + integration suite and gates every changed line; without it the job runs DB-free and still gates the pure modules (`app/trust/**`, `app/components/**`, `app/listings/**`, `db/schema.ts`), excluding only `app/server/**` (which needs the DB to be covered) and logging that exclusion explicitly. Zero new dependencies; the pure parsers/intersector are unit-tested in `tests/unit/diff-coverage.test.ts`.
