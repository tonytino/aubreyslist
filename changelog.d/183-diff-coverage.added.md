- `[manual]` Added a diff-coverage CI gate that hard-fails a PR when its added/changed executable lines are covered below 80% (`.github/scripts/check-diff-coverage.mjs`, issue #183). Coverage is measured only on the lines the PR touches, so legacy untested code is never retroactively failed; the job runs the full unit + integration suite under the same `CI_E2E_DATABASE_URL` secret gate as `integration-e2e` (and self-skips when the secret is absent) so DB-only `app/server/**` lines are not misread as uncovered. Zero new dependencies; the pure parsers/intersector are unit-tested in `tests/unit/diff-coverage.test.ts`.
