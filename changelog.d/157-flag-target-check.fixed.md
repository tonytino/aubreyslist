- `[manual]` Hardened the moderation write layer (`applyModerationAction`) to verify a supplied `flagId` actually targets the content being acted on: before any write it now SELECTs the flag's exclusive-arc target (`listingId`/`claimId`/`incidentId`) and rejects a mismatch (or not-found) with a `422`, so a moderator (or crafted call) can no longer close an unrelated flag and write an internally inconsistent audit row. Direct actions without a `flagId`, and flags that match, behave exactly as before (closes #157).
