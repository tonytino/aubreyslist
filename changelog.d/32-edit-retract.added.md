- `[manual]` Added "edit/retract your own contributions" (#32, EPIC 4 / Trust, domain.md → Roles & Permissions). Incidents: new owner-only `editIncident` (occurredOn/severity/note) and `retractIncident` (delete) server functions in `app/server/incidents/` (client-callable as `updateIncident` / `removeIncident` via `incidents.fn.ts`), login-gated and rate-limited like the existing report path. Ownership is enforced SERVER-SIDE — both mutations match on `(id, userId = current user)` and reject with 403 when zero rows are affected, so a non-owner (or anonymous) caller can never touch someone else's report; it is never just a hidden UI control. The edit reuses the exact `occurredOn` validation from the report schema (real, non-future calendar date), so editing a date in/out of the 90-day window correctly raises or drops the recent-incident banner, and retracting a recent incident clears it — recency/aggregates recompute from fresh evidence on the next read. Attestations: the change/retract logic already existed (`castVote` upsert + `retractVote` delete on the one-vote-per-user constraint, #28); this wires it into the listing-detail "Community claims" surface with per-claim Confirm/Dispute/Retract controls (`ClaimVoteControls`) shown only to signed-in viewers, highlighting the viewer's own vote (the roll-up loader now returns `viewerVote` per claim). On the listing detail page (`app/routes/listings.$id.tsx`) the claim roll-up moved to a TanStack Query so a vote change/retract invalidates it and the counts, recency, own-vote highlight, and headline celiac-safe cue all update; incident edit/retract controls appear inline only on the viewer's own reports with a confirm-before-delete step, and invalidate the incident list + banner. Client/server boundary stays clean for the production build: the attestation `createServerFn` wrappers were extracted to `app/server/attestations/attestations.fn.ts` (the `*.fn.ts` convention) with a client-safe `attestation_value` mirror, so no `getDb`/drizzle code enters the browser bundle. Moderators removing ANY content remains out of scope (EPIC 6).
