- `[manual]` Added trust/recency/alphabetical sort to the browse list (#36, domain.md → Discovery). A new accessible, labeled `Sort by` `<select>` on `/listings` is URL-driven via `?sort=` (mirroring the `?page=` pattern: linkable, back/forward works, and changing the sort resets to page 1); selection — never colour — conveys state. Three options: **Alphabetical** (name A–Z, the stable default), **Most trusted**, and **Recently confirmed**. CRITICAL for safety: "Most trusted" orders by the SAME safety tier the card displays — celiac-safe (fresh, uncontested confirm-majority) > may-be-stale > contested/gluten-friendly > unattested — reproducing `deriveHeadlineSafetyState` in SQL over the same visible signals (confirm/dispute counts + `lastConfirmedAt` staleness, ADR-007), so a listing the product flags as stale or contested can never outrank a fresh celiac-safe one; within a tier it orders by net confirms, then most-recently-confirmed, then name. "Recently confirmed" orders most-recently-confirmed first, then net confirms, then name. The SQL staleness cutoff is threaded from the same `now`/`staleness_months` the displayed glance uses, so the sort boundary matches the card exactly (no drift); it is a roll-up of visible evidence, never an opaque score. Recent incidents deliberately do not influence the v1 sort (the card's incident flag stays visible; incident-demotion in ordering is a later issue). Sort is COMBINABLE with text search (#34): the search predicate is now threaded through the browse loader's `WHERE` (applied to both the page and total-count queries so pagination/total stay correct) while sort only changes `ORDER BY`. The sort set lives in a single extensible registry (`app/listings/sort.ts`, client-safe pure) shared by the control and the loader, and the safety tier is a pure `safetyTierRank` (`app/trust/summary.ts`) derived from the displayed state so the sort can't drift from the glance — so #37's "near me" `distance` sort is a clean addition (one registry entry + one `ORDER BY` branch), not a rewrite. Unit tests cover the registry/parser, the safety-tier ranking over a mixed set (fresh-safe ranks above stale-high-net and contested — the regression guard), the per-sort `ORDER BY`, the staleness-cutoff threading, and search+sort+pagination combination; the browse Playwright smoke test covers the labeled control and its `?sort=` URL wiring.
