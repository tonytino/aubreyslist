- `[manual]` Fixed the recent-incident banner never rendering on the listing-detail page (and the incident date formatting) when reading from the live database. `incidents.occurred_on` is a Postgres `date` (Drizzle `PgDateString`, which passes the driver value through verbatim), but the Neon HTTP driver returns a `date` as a JS `Date` rather than a `YYYY-MM-DD` string — so the recency logic (`parseCalendarDay`) silently failed. Normalized `occurredOn` to a canonical `YYYY-MM-DD` at the read boundary (new `toCalendarDayString` helper, applied in `listIncidents`/`reportIncident`/`editIncident` and the browse recent-incident signal), keeping the calendar-date contract. Added a `tests/integration` round-trip test (real Postgres) and unit tests as regression guards.
