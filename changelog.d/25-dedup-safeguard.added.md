- `[manual]` Hardened the add-listing duplicate safeguard (issue #25, ADR-008) in `app/server/listings/create.ts`: a Places-mode submission for an already-listed Place ID resolves to the existing listing (`created: false`) instead of erroring, and a manual-mode submission is now blocked when it matches an existing **visible** manual listing on a normalized name+address (trim, collapse whitespace, lowercase, fold diacritics, strip punctuation — `app/server/listings/dedup.ts`). Moderator-hidden/removed listings are excluded from the candidate set so they can't wrongly block a re-add or point a user at a row they can't see. The block raises a structured `DuplicateListingError` carrying the existing listing's id/name; the manual intake form (`app/components/add-listing/ManualIntakeForm.tsx`) surfaces a link straight to that existing listing.
